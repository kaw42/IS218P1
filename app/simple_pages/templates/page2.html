About Docker

Docker is an open source containerization platform and application build and deployment tool. It is an open platform for developing, shipping, and running applications through containers.

What are containers?

A container is a standard unit of software that packages up code and all its dependencies so the application runs quickly and reliably from one computing environment to another.

What are images?

An image is a lightweight, standalone, executable package of software that includes everything needed to run an application: code, runtime, system tools, system libraries and settings.

Link to the project repository on docker hub

Docker Commands

docker build = Build an image from a Dockerfile
docker commit = Create a new image from a container’s changes
docker container = manage containers
docker kill = kill running containers
docker push = push an image or repo to a registry
docker rm = remove containers
docker run = run a command in a new container

How to start the project and access the container with the terminal

Open a terminal and go to the app directory with your docker file. You can build a container using the docker "build" command. ex.) docker build -t container-name. This command uses the docker file to build a new container image and the "-t" flag tags your image.

To access the container, you have to use the "docker run" command. ex.) docker run -dp 3000:3000 container_name. The 3000:3000 means you are creating a mapping between the host’s port 3000 to the container’s port 3000. Now when you go to your browser and enter http://localhost:3000, you should be able to see your app.
